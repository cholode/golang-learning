### builder模式

定义一堆成员变量，然后返回的时候返回结构体本身就可以嵌套访问成员变量了

```go
type selects struct{
    From string,
    Where string,
    Offset int,
    Oimit int,
    Order string,
}

func (sb *selects) Order *selects{} 

var sb *selects
sb.Select("name").
    From("student").
    Where("Score>80").
    Offset(10)
//可以这么嵌套调用


int
```

### 外观模式

将多个个不同的系统放在同一个结构体里进行统一的使用，就是对系统方法的封装

```go
type Mysql struct{}

func (Mysql) Update() {}

type Redis struct{}

func (Redis) Update() {}

type Store struct {
    DB *Mysql
    Cache *Redis
}

func (s Store) Update() {//调用一个update就相当于调用了两个系统的update
    s.DB.Update()
    s.Cache.Update()
}


int
```

### 代理模式

对于原来的方法，可以通过代理模式进行功能的增强

```go
type Mysql struct{}

func (Mysql) Query() any {}//原本有的接口

type Store struct {
    DB    *Mysql
    Cache *Redis
}

func (s Store) Query() any {
    data := s.Cache.Query()//增加新的查缓存的功能
    if data != nil {
        return data
    }
    data = s.DB.Query()//调用原来的接口

    return data
}
```
